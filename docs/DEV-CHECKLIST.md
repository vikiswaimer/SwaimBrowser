# Чеклист разработчика

Обязательные проверки перед каждым коммитом.

> **Скоуп v0.1:** см. [PRD.md](./PRD.md) — что делаем и что НЕ делаем  
> **Терминология:** см. [GLOSSARY.md](./GLOSSARY.md)  
> **Известные проблемы:** см. [KNOWN-ISSUES.md](./KNOWN-ISSUES.md)

---

## Перед коммитом

### 1. Код

- [ ] Код запускается без ошибок в консоли
- [ ] Нет `console.log` отладочных сообщений
- [ ] Функции имеют осмысленные имена (см. глоссарий)
- [ ] Новые переменные/функции добавлены в GLOSSARY.md

### 2. Функциональность

- [ ] **Focus Mode:** таймер корректно запускается/останавливается
- [ ] **Focus Mode:** overlay появляется и скрывается
- [ ] **Блокировка:** заблокированные сайты детектируются (см. список в глоссарии)
- [ ] **Инсайты:** сохраняются и отображаются в списке
- [ ] **Навигация:** Back/Forward/Reload работают

### 3. Горячие клавиши

| Комбинация | Проверить |
|------------|-----------|
| Alt+P | Открывается диалог добавления инсайта |
| Ctrl+Shift+F | Включается/выключается Focus Mode |
| Enter в URL | Навигация или поиск работает |

### 4. UI/UX

- [ ] Sidebar открывается/закрывается
- [ ] Вкладки Focus/Projects/Insights переключаются
- [ ] Кнопки имеют hover-состояние
- [ ] Цвета соответствуют CSS-переменным (см. глоссарий)

### 5. Данные

- [ ] Формат инсайта соответствует спецификации:
  ```javascript
  { url, text, timestamp, tags }
  ```
- [ ] Данные сохраняются между перезапусками (через store)

---

## При изменении горячих клавиш

- [ ] Обновить таблицу в GLOSSARY.md
- [ ] Проверить конфликты с системными сочетаниями
- [ ] Проверить на всех ОС (если возможно)

---

## При добавлении новой функции

1. [ ] **Сверить с PRD.md** — функция входит в scope v0.1?
2. [ ] Добавить термины в GLOSSARY.md
3. [ ] Если есть ограничения — задокументировать в KNOWN-ISSUES.md
4. [ ] Проверить, не сломана ли существующая функциональность
5. [ ] Обновить CSS-переменные, если добавлены новые цвета

> **Важно:** Если функция НЕ входит в scope v0.1, добавьте её в [трекер отклонённых идей](./PRD.md#7-трекер-отклонённых-идей)

---

## При изменении формата данных

- [ ] Миграция старых данных (если store уже использовался)
- [ ] Обновить формат в GLOSSARY.md
- [ ] Проверить `loadInsights()` и `addInsight()`

## Известные ловушки

> Детали в [KNOWN-ISSUES.md](./KNOWN-ISSUES.md)

| Ловушка | Как избежать |
|---------|--------------|
| NW.js + Electron конфликт | Не смешивать API разных платформ |
| Блокировка после загрузки | Помнить, что `blockNavigation` срабатывает на `dom-ready` |
| Теги инсайтов пустые | Не полагаться на `tags[]` до реализации |
| Event listener leak | Проверять удаление слушателей в `endFocusMode()` |

```
✅ Открыть index.html в среде
✅ Ctrl+Shift+F → Focus Mode включается
✅ Esc или кнопка → Focus Mode выключается

✅ Ввести текст → инсайт появляется в списке
✅ 📋 кнопка → sidebar открывается
✅ Вкладки переключаются
```

## Коммит

После прохождения чеклиста:

```bash
git add .
git commit -m "feat/fix/refactor: краткое описание"
```

**Формат коммита:**
- `feat:` — новая функция
- `fix:` — исправление бага
- `refactor:` — рефакторинг без изменения поведения
- `docs:` — изменения документации
- `style:` — форматирование, CSS
