# Product Requirements Document: Swaim Browser v0.1

> **Цель документа:** Фиксация скоупа MVP для предотвращения расползания фич.  
> **Версия:** v0.1  
> **Дата:** 2026-02-24

---

## 1. Целевой пользователь

### Персона: Продакт-менеджер / UX-исследователь

**Кто это:**
- Работает с веб-продуктами
- Ежедневно проводит исследования в браузере
- Борется с прокрастинацией и отвлечениями
- Накапливает инсайты из разных источников
- Работает над несколькими проектами параллельно

**Боли:**
1. Постоянные отвлечения на соцсети во время рабочих сессий
2. Потеря контекста при переключении между исследовательскими задачами
3. Инсайты теряются или разбросаны по разным приложениям
4. Сложно поддерживать дисциплину глубокой работы

**Потребности:**
- Среда для концентрированной работы
- Быстрое сохранение мыслей без переключения контекста
- Блокировка отвлекающих ресурсов

---

## 2. Цели v0.1

### Бизнес-цели
1. Валидировать гипотезу: продакт-менеджерам нужен специализированный браузер
2. Собрать 10-20 активных тестировщиков для обратной связи
3. Определить ключевую метрику (DAU, количество focus-сессий)

### Продуктовые цели
1. Работающий прототип с core-функциями
2. Возможность провести полноценную focus-сессию
3. Возможность сохранить инсайт во время исследования

### Технические цели
1. Выбор и унификация платформы (NW.js ИЛИ Electron)
2. Базовая архитектура для дальнейшего развития
3. Локальное хранение данных (без серверной части)

## 5. Чеклист для новых идей

Перед добавлением любой новой фичи задайте вопросы:

### Критерии включения в v0.1
- [ ] Без этого невозможно провести базовую focus-сессию?
- [ ] Без этого невозможно сохранить хотя бы один инсайт?
- [ ] Это критичный баг, блокирующий использование?

### Красные флаги (STOP)
- ❌ "Было бы круто добавить..."
- ❌ "Конкуренты делают так..."
- ❌ "Потом будет сложнее..."
- ❌ "Это займёт всего пару часов..."
- ❌ Требует серверной части
- ❌ Требует внешних API
- ❌ Требует дополнительных платформ

### Решение
1. **Да на все критерии** → Добавляем в v0.1
2. **Нет хотя бы на один** → Записываем в backlog, ревизия после v0.1

## 7. Технические ограничения v0.1

### Платформа
- **Решить:** NW.js или Electron (не оба)
- Текущий статус: смешанные API (см. KNOWN-ISSUES.md)

### Хранение данных
- Только локальное (electron-store или аналог)
- Без облачной синхронизации
- Без авторизации

### Поддерживаемые ОС
- Только desktop (Windows/macOS/Linux)
- Без мобильных версий

## 8. Критерии готовности v0.1

### Must Have (без этого не релизим)
- [ ] Приложение запускается без ошибок
- [ ] Focus Mode работает полный цикл (старт → таймер → завершение)
- [ ] Можно сохранить инсайт через Alt+P
- [ ] Заблокированные сайты не открываются в Focus Mode
- [ ] Данные сохраняются между сессиями
- [ ] Выбрана одна платформа (NW.js или Electron)

### Should Have (желательно)
- [ ] Блокировка ДО загрузки страницы
- [ ] Inline-форма для инсайтов (вместо prompt)
- [ ] Звук по окончании сессии

### Nice to Have (можно позже)
- [ ] Красивые анимации
- [ ] Кастомная иконка приложения

## 9. Метрики v0.1

### Качественные
- Можно провести 25-минутную focus-сессию от начала до конца
- Сохранённые инсайты не теряются
- Отвлекающие сайты блокируются

### Количественные (для тестирования)
- Количество focus-сессий на пользователя в день
- Количество сохранённых инсайтов
- Количество попыток открыть заблокированный сайт
