> **Версия:** 0.2  
> **Статус:** Draft  
> **Последнее обновление:** 2026-03-16  
> **Цель документа:** Фиксация скоупа MVP с интегрированным Product Framework для управления бизнесом

---

## 1. Видение продукта

**Swaim Browser** — специализированный браузер для предпринимателей и продакт-менеджеров, который объединяет глубокую фокусировку (Deep Work) с полноценным продуктовым фреймворком для управления бизнесом.

### Миссия

Помочь digital nomads и стартаперам работать осознанно: не просто «сёрфить интернет», а систематически исследовать, фиксировать находки, управлять гипотезами и превращать их в бизнес-результаты.

### Уникальное ценностное предложение (UVP)

> Единственный браузер, который совмещает **Deep Work Mode** для глубокой концентрации с **Digital Business Space** — полноценным фреймворком для управления малым бизнесом и стартапом в одном окне.

### Ключевое отличие

В отличие от обычных браузеров, Swaim:
- **Не просто блокирует отвлечения** — помогает достигать целей
- **Не просто сохраняет закладки** — структурирует знания и гипотезы
- **Не просто отображает страницы** — интегрирует исследования с бизнес-решениями

## 2. Целевой пользователь

### 2.1 Первичная аудитория: Digital Nomad + Lean Startup

| Характеристика | Описание |
|----------------|----------|
| **Кто** | Предприниматель-одиночка или маленькая команда (1-5 человек) |
| **Где работает** | Удалённо: кофейни, коворкинги, дом, путешествия |
| **Как работает** | Lean Startup: гипотезы → эксперименты → learning |
| **Боль #1** | Постоянные отвлечения (соцсети, YouTube) убивают глубокую работу |
| **Боль #2** | Инсайты из исследований теряются в хаосе вкладок и заметок |
| **Боль #3** | Нет единой системы для связи исследований с продуктовыми решениями |
| **Боль #4** | Управление бизнесом размазано по 10+ инструментам |

### 2.2 Персона: Алекс

**Алекс, 32 года, digital nomad, строит SaaS-продукт**

- Живёт в Лиссабоне (сейчас), работает из коворкинга
- Запустил MVP, проводит customer discovery
- Каждый день: 50% исследования, 30% общение с клиентами, 20% разработка
- Использует: Notion (хаотично), Chrome (100+ вкладок), Pomodoro-приложение
- Проблема: открывает Twitter «на минутку» — теряет 40 минут
- **Мечта:** Один инструмент, где видение, гипотезы, исследования и задачи связаны

**Цитата:**
> «Мне нужен браузер, который не только помогает НЕ отвлекаться, но и структурирует мой бизнес прямо там, где я работаю.»

### 2.3 Jobs to Be Done

| Job | Описание | Частота |
|-----|----------|---------|
| **Deep Work Session** | Сфокусироваться на задаче без отвлечений | 3-5 раз в день |
| **Capture Insight** | Быстро записать мысль/боль/идею с контекстом | 10-20 раз в день |
| **Manage Hypotheses** | Создать, отслеживать и валидировать гипотезы | 1-3 раза в день |
| **Track Metrics** | Видеть ключевые метрики и прогресс | 1-2 раза в день |
| **Review & Connect** | Просмотреть инсайты, связать с гипотезами и задачами | 1-2 раза в день |
| **Block Distractions** | Не попасть на отвлекающие сайты | Постоянно |
| **Plan & Execute** | Управлять задачами и спринтами | Ежедневно |

## 3. Цели продукта

### 3.1 Бизнес-цели v0.2

| Цель | Метрика | Таргет |
|------|---------|--------|
| Валидация концепции | # активных пользователей | 200 DAU |
| Retention | % пользователей через 7 дней | 45% |
| Engagement | Средние Focus сессии в день | 2.5+ |
| Framework Adoption | % пользователей с заполненным Vision | 60% |
| Product-market fit | NPS | 45+ |

### 3.2 Пользовательские цели

1. **Глубокая концентрация** — провести рабочую сессию без отвлечений
2. **Структурированный сбор** — не терять инсайты из исследований
3. **Управление гипотезами** — системно проверять идеи
4. **Прозрачность метрик** — видеть прогресс бизнеса
5. **Связь с продуктом** — видеть как инсайты влияют на решения

### 3.3 Ключевые метрики (North Star)

**Primary:** Количество валидированных гипотез с связанными инсайтами

**Supporting:**
- Время в Focus Mode / общее время
- Количество сохранённых инсайтов
- % гипотез с привязанными экспериментами
- % сессий с заполненной целью

#### 4.2.1 VISION — Видение

**Назначение:** Фиксация долгосрочного направления бизнеса

| Поле | Описание | Пример |
|------|----------|--------|
| **Mission** | Зачем существует компания | «Помогаем предпринимателям работать осознанно» |
| **Vision** | Каким будет мир через 5-10 лет | «Каждый стартапер использует наш браузер для исследований» |
| **Values** | Ключевые ценности (3-5) | Focus, Simplicity, Learning |
| **Big Hairy Goal** | Амбициозная цель | «1M пользователей к 2028» |

**Как применяется в браузере:**
- Отображается в дашборде при запуске
- Напоминание о миссии перед каждой Focus-сессией
- Проверка: «Соответствует ли эта задача нашему видению?»

**Статус:** ⚠️ Требует реализации

#### 4.2.2 STRATEGY — Стратегия

**Назначение:** Квартальные и годовые цели в формате OKR

| Компонент | Описание |
|-----------|----------|
| **Objectives** | Качественные цели на период |
| **Key Results** | Измеримые результаты (3-5 на Objective) |
| **Initiatives** | Конкретные проекты для достижения |

**Структура данных:**

```typescript
interface OKR {
  id: string;
  objective: string;
  period: 'Q1' | 'Q2' | 'Q3' | 'Q4' | 'Year';
  year: number;
  keyResults: KeyResult[];
  progress: number; // 0-100%
}

interface KeyResult {
  id: string;
  description: string;
  currentValue: number;
  targetValue: number;
  unit: string;
  linkedTasks: string[]; // Task IDs
}
```

**Как применяется в браузере:**
- Виджет прогресса OKR в sidebar
- При создании задачи — привязка к Key Result
- Еженедельный review прогресса

**Статус:** ⚠️ Требует реализации

#### 4.2.5 HYPOTHESES — Гипотезы

**Назначение:** Системное управление гипотезами по методологии Lean Startup

| Поле | Описание |
|------|----------|
| **Statement** | Формулировка гипотезы |
| **Type** | Тип: Problem, Solution, Channel, Revenue |
| **Assumption** | Ключевое допущение |
| **Risk Level** | High / Medium / Low |
| **Status** | Draft, Testing, Validated, Invalidated |
| **Evidence** | Связанные инсайты и эксперименты |

**Структура данных:**

```typescript
interface Hypothesis {
  id: string;
  statement: string;
  type: 'problem' | 'solution' | 'channel' | 'revenue' | 'other';
  assumption: string;
  riskLevel: 'high' | 'medium' | 'low';
  status: 'draft' | 'testing' | 'validated' | 'invalidated';
  experiments: string[]; // Experiment IDs
  insights: string[]; // Insight IDs
  createdAt: string;
  updatedAt: string;
  validatedAt?: string;
}
```

**Как применяется в браузере:**
- Список гипотез в sidebar
- При исследовании — быстрая привязка инсайта к гипотезе (Alt+H)
- Kanban-доска гипотез по статусу
- Напоминание: «Какую гипотезу проверяем сегодня?»

**Статус:** ⚠️ Требует реализации | P0
