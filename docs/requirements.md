# Функциональные требования

## Обзор

Swaim Browser — продуктивный браузер для исследователей, продакт-менеджеров и стартаперов. Цель: помочь сфокусироваться на работе и систематизировать инсайты.

## Модули и требования

---

## 1. Модуль навигации (Navigation)

### FR-NAV-001: Ввод URL
- **Описание:** Пользователь может ввести URL в адресную строку и перейти по нему
- **Элемент:** `#urlInput`
- **Триггер:** Нажатие Enter
- **Поведение:**
  - Если введён валидный URL (содержит `.`) → добавить `https://` если нет протокола
  - Если не URL → выполнить поиск Google
- **Критерии приёмки:**
  - [x] `example.com` → `https://example.com`
  - [x] `http://example.com` → без изменений
  - [x] `test query` → `https://www.google.com/search?q=test+query`

### FR-NAV-002: Навигация назад
- **Описание:** Переход на предыдущую страницу в истории
- **Элемент:** `#backBtn`
- **Триггер:** Клик
- **Поведение:** Вызов `webview.goBack()`

### FR-NAV-003: Навигация вперёд
- **Описание:** Переход на следующую страницу в истории
- **Элемент:** `#forwardBtn`
- **Триггер:** Клик
- **Поведение:** Вызов `webview.goForward()`

### FR-NAV-004: Перезагрузка страницы
- **Описание:** Перезагрузка текущей страницы
- **Элемент:** `#reloadBtn`
- **Триггер:** Клик
- **Поведение:** Вызов `webview.reload()`

### FR-NAV-005: Синхронизация URL
- **Описание:** Адресная строка отображает текущий URL
- **Триггер:** Событие `dom-ready` на webview
- **Поведение:** `urlInput.value = webview.src`

## 2. Модуль фокусировки (Focus Mode)

### FR-FOC-001: Запуск сессии фокусировки
- **Описание:** Пользователь может запустить таймер фокусировки
- **Элементы:** `#focusBtn`, `#startFocusSession`
- **Триггер:** Клик по кнопке или `Ctrl+Shift+F`
- **Поведение:**
  1. Активировать `#focusOverlay`
  2. Запустить обратный отсчёт
  3. Включить блокировку отвлекающих сайтов
- **Критерии приёмки:**
  - [x] Оверлей становится видимым
  - [x] Таймер отсчитывает время
  - [x] Кнопка меняет иконку на ⏸️

### FR-FOC-002: Таймер обратного отсчёта
- **Описание:** Отображение оставшегося времени в формате MM:SS
- **Элемент:** `#focusTimer`
- **Поведение:**
  - Обновление каждую секунду
  - Формат: `MM:SS` с ведущими нулями для секунд
- **Критерии приёмки:**
  - [x] `25:00` → `24:59` → ... → `0:00`
  - [x] При достижении `0:00` сессия автоматически завершается

### FR-FOC-003: Блокировка отвлекающих сайтов
- **Описание:** Во время сессии переходы на отвлекающие сайты блокируются
- **Заблокированные домены:**
  - youtube.com
  - twitter.com
  - reddit.com
  - facebook.com
  - instagram.com
- **Поведение:**
  - При попытке перехода показывается сообщение в `#blockedSites`
  - Формат: `🚫 domain.com blocked during focus session`

### FR-FOC-004: Завершение сессии
- **Описание:** Пользователь может досрочно завершить сессию
- **Элемент:** `#endFocusBtn`
- **Триггер:** Клик
- **Поведение:**
  1. Остановить таймер
  2. Скрыть оверлей
  3. Отключить блокировку сайтов
  4. Вернуть иконку кнопки к ⚡

### FR-FOC-005: Выбор длительности сессии
- **Описание:** Пользователь может выбрать длительность перед стартом
- **Элемент:** `#sessionDuration`
- **Варианты:**
  - 25 минут (по умолчанию)
  - 50 минут
  - 90 минут

### FR-FOC-006: Цель сессии
- **Описание:** Пользователь может указать цель сессии
- **Элемент:** `#sessionGoal`
- **Поведение:** Текст отображается в `#focusGoal` во время сессии

## 3. Модуль сайдбара (Sidebar)

### FR-SB-001: Открытие/закрытие сайдбара
- **Описание:** Пользователь может показать/скрыть боковую панель
- **Элементы:** `#sidebarBtn`, `#closeSidebar`
- **Триггер:** Клик
- **Поведение:**
  - Toggle класса `.open` на `#sidebar`
  - Анимация slide-in/slide-out (300ms)

### FR-SB-002: Переключение вкладок
- **Описание:** Навигация между вкладками сайдбара
- **Элементы:** `.tab-btn`
- **Вкладки:**
  - Focus — настройки фокусировки
  - Projects — управление проектами
  - Insights — список инсайтов
- **Поведение:**
  - Активная вкладка получает класс `.active`
  - Соответствующий `.tab-pane` становится видимым

## 4. Модуль инсайтов (Insights)

### FR-INS-001: Добавление инсайта
- **Описание:** Пользователь может сохранить заметку с привязкой к текущей странице
- **Триггер:** `Alt+P` или кнопка `#addInsightBtn`
- **Поведение:**
  1. Открыть `prompt()` для ввода текста
  2. Сохранить в store с полями:
     - `url` — текущий URL webview
     - `text` — введённый текст
     - `timestamp` — ISO timestamp
     - `tags` — пустой массив
  3. Обновить список инсайтов

### FR-INS-002: Отображение инсайтов
- **Описание:** Список последних инсайтов в сайдбаре
- **Элемент:** `#insightsList`
- **Поведение:**
  - Показывать последние 5 инсайтов
  - Формат: текст инсайта в блоке с border-bottom

### FR-INS-003: Персистентность
- **Описание:** Инсайты сохраняются между сессиями
- **Хранилище:** `electron-store`, ключ `insights`
- **Формат:** см. [DATA.md](./DATA.md)

### FR-HK-002: Переключить Focus Mode
- **Комбинация:** `Ctrl + Shift + F`

## Нефункциональные требования

### NFR-001: Производительность
- Время отклика UI: < 100ms
- Загрузка приложения: < 2s

### NFR-002: Доступность данных
- Данные должны сохраняться локально
- Работа offline без ограничений

### NFR-003: Кроссплатформенность
- Поддержка: Windows, macOS, Linux
- Фреймворк: NW.js / Electron

### NFR-004: Дизайн
- Тёмная тема по умолчанию
- Современный минималистичный UI
- Цветовая схема: см. CSS-переменные в [ARCHITECTURE.md](./ARCHITECTURE.md)
