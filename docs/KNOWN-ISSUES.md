# Известные ограничения и технический долг

Документ описывает текущие ограничения, костыли и планы по их устранению.

> **Терминология:** см. [GLOSSARY.md](./GLOSSARY.md)

## Критичные ограничения

### 1. Смешение NW.js и Electron API

**Проблема:** Код использует `require('nw.gui')` и `electron-store` одновременно, что невозможно в реальности.

**Где:** `index.html`, строки 324-326

```javascript
const nw = require('nw.gui');
const Store = require('electron-store');
```

**Влияние:** Приложение не запустится без выбора одной платформы.

**План:** Выбрать платформу (NW.js или Electron) и унифицировать API.

---

### 2. Блокировка сайтов работает только после загрузки

**Проблема:** `blockNavigation()` срабатывает на `dom-ready`, т.е. страница уже загружена.

**Где:** `index.html`, функция `blockNavigation()`

**Влияние:** Пользователь видит контент заблокированного сайта до появления оверлея.

**План:** Перехватывать навигацию на этапе `will-navigate`.

---

### 3. Теги инсайтов не используются

**Проблема:** Поле `tags: []` всегда пустое, UI для тегов отсутствует.

**Где:** Функция `addInsight()`, формат инсайта

**Влияние:** Невозможно категоризировать инсайты.

**План:** Добавить UI для тегов или автоматическую классификацию.

---

## Ограничения UX

### 4. Prompt для ввода инсайта

**Проблема:** Используется `prompt()`, который:
- Блокирует весь UI
- Не поддерживает многострочный текст
- Не позволяет отменить без потери данных

**Где:** `addInsight()`, строка 459

**Влияние:** Плохой UX при частом сохранении инсайтов.

**План:** Заменить на inline-форму или floating widget.

---

### 5. Показываются только 5 последних инсайтов

**Проблема:** `.slice(-5)` ограничивает отображение.

**Где:** `loadInsights()`, строка 472

**Влияние:** Нет доступа к старым инсайтам через UI.

**План:** Добавить пагинацию или infinite scroll.

---

### 6. Focus таймер без восстановления

**Проблема:** При перезагрузке страницы таймер сбрасывается.

**Где:** `focusTimeLeft = 25 * 60` — жёстко заданное значение.

**Влияние:** Потеря прогресса сессии при случайной перезагрузке.

**План:** Сохранять состояние таймера в store.

---

## Технический долг

### 7. Нет разделения кода

**Проблема:** Весь JS в одном `<script>` блоке внутри HTML.

**Где:** `index.html`, строки 323-483

**Влияние:**
- Сложно тестировать
- Нет code reuse
- Сложно масштабировать

**План:** Вынести в отдельные модули: `focus.js`, `insights.js`, `navigation.js`.

---

### 8. CSS без препроцессора

**Проблема:** Inline styles в HTML, без CSS-модулей или препроцессора.

**Где:** `<style>` блок, строки 6-238

**Влияние:** Сложно поддерживать при росте проекта.

**План:** Вынести в `styles.css`, рассмотреть Tailwind или CSS Modules.

---

### 9. Нет валидации URL

**Проблема:** URL вводится без валидации, только базовые проверки.

**Где:** `urlInput keypress` handler

```javascript
if (!url.startsWith('http')) {
  if (url.includes('.')) url = 'https://' + url;
  else url = 'https://www.google.com/search?q=' + ...
}
```

**Влияние:** Возможны ошибки навигации, XSS через URL.

**План:** Добавить proper URL validation и sanitization.

---

### 10. Event listener не удаляется корректно

**Проблема:** `webview.addEventListener('dom-ready', blockNavigation)` добавляется, но при `endFocusMode()` слушатель может не удалиться из-за binding.

**Где:** `startFocusMode()` / `endFocusMode()`

**Влияние:** Memory leak, неожиданное поведение блокировки.

**План:** Использовать AbortController или именованные функции с корректным удалением.

|---------|--------|-----------|
| Настройка списка заблокированных сайтов | Не реализовано | Высокий |
| История Focus сессий | UI есть, логика нет | Высокий |
| Управление проектами | Только UI-заглушка | Средний |
| Экспорт инсайтов | Не реализовано | Средний |

| Платформа | Статус | Примечание |
