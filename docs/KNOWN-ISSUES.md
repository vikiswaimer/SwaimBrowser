## Ограничения NW.js 0.92

### Архитектурные

| Ограничение | Влияние |
|-------------|---------|
| Размер (~150-200 MB) | Большой дистрибутив |
| Память (300+ MB базово) | Высокое потребление RAM |
| Холодный старт (2-5 сек) | Задержка при запуске |

### Платформенные

- **macOS ARM** — требуется Rosetta 2 или отдельная сборка
- **Linux** — зависимости от GTK, libnss3, libatk
- **Windows ARM** — не поддерживается

### API ограничения

- Webview использует legacy `<webview>` с рисками безопасности
- Chrome Extensions **не поддерживаются**
- Версия Chromium может отставать от stable

## Осознанные решения "не делаем"

| Функция | Причина | Пересмотреть когда |
|---------|---------|-------------------|
| Автообновления | Сложность, инфраструктура, code signing | 1000+ пользователей |
| Mobile/Web | NW.js — desktop технология | Валидация PMF |
| Облачная синхронизация | Приватность, отсутствие backend | Team features |
| Multi-profile | Усложнение UX, низкий приоритет | Запрос от пользователей |
| Chrome Extensions | NW.js не поддерживает | Решение от NW.js |

## Известные проблемы

### Критические

| ID | Проблема | Workaround |
|----|----------|------------|
| KB-001 | electron-store в NW.js (shim может сломаться) | Мониторинг при обновлении |

### Высокий приоритет

| ID | Проблема | Workaround |
|----|----------|------------|
| KB-002 | Webview nodeintegration (риски безопасности) | Не загружать недоверенный контент |
| KB-003 | Focus блокировка неполная (overlay можно закрыть) | — |

### Средний приоритет

| ID | Проблема | Workaround |
|----|----------|------------|
| KB-004 | Монолитный index.html | Затрудняет поддержку |
| KB-005 | Hardcoded blocked domains | Требует пересборки |
| KB-006 | prompt() для инсайтов — плохой UX | Заменить на inline UI |

### Низкий приоритет

| ID | Проблема |
|----|----------|
| KB-007 | Нет обработки ошибок загрузки |
| KB-008 | Нет визуальной индикации истории навигации |
| KB-009 | Таймер сбрасывается при закрытии |
