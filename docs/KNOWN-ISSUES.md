# Известные ограничения и технический долг

Этот документ описывает текущие ограничения, осознанные решения "не делаем", известные проблемы и планы по рефакторингу проекта Swaim Browser.

> **Цель документа:** предотвратить повторные предложения решений, которые уже были рассмотрены и отклонены или отложены.

---

## 📦 Ограничения NW.js 0.92

### Архитектурные ограничения

| Ограничение | Описание | Влияние |
|-------------|----------|---------|
| **Размер приложения** | NW.js включает полный Chromium (~150-200 MB минимум) | Большой размер дистрибутива, долгая загрузка |
| **Память** | Chromium потребляет значительный объём RAM | Высокое потребление памяти (300+ MB базово) |
| **Холодный старт** | Запуск Chromium занимает время | 2-5 секунд до готовности приложения |

### Платформенные ограничения

| Платформа | Ограничение |
|-----------|-------------|
| **macOS ARM (Apple Silicon)** | Требуется Rosetta 2 для x64 билдов или отдельная сборка для ARM |
| **Linux** | Зависимости от системных библиотек (GTK, libnss3, libatk и др.) |
| **Windows** | Нет нативной поддержки Windows on ARM |

### API ограничения NW.js

- **Webview** — используется legacy `<webview>` тег с атрибутом `nodeintegration`, что несёт риски безопасности
- **Native file dialogs** — работают, но с ограничениями в некоторых сценариях
- **Tray icon** — поддержка есть, но поведение отличается между ОС
- **Notifications** — нативные уведомления работают не на всех системах одинаково

### Chromium-специфичные ограничения

- Некоторые экспериментальные Web API могут быть отключены
- Версия Chromium в NW.js 0.92 может отставать от актуальной stable версии
- Расширения Chrome **не поддерживаются** напрямую

## 🚫 Осознанные решения "не делаем"

Следующие функции были рассмотрены и сознательно отложены или исключены:

### Автообновления

| Решение | Не реализуем автоматические обновления |
|---------|----------------------------------------|
| **Причина** | Сложность реализации, необходимость инфраструктуры для хостинга обновлений, проблемы с code signing |
| **Альтернатива** | Ручная загрузка новых версий с сайта |
| **Пересмотреть когда** | Появится стабильная user base (1000+ пользователей) |

### Кроссплатформенная сборка с одной машины

| Решение | Сборка каждой платформы на своей ОС |
|---------|-------------------------------------|
| **Причина** | Code signing для macOS требует macOS, Wine для Windows билдов нестабилен |
| **Альтернатива** | CI/CD с несколькими runners (macOS, Windows, Linux) |
| **Пересмотреть когда** | Настроим GitHub Actions с matrix builds |

### Mobile/Web версия

| Решение | Только desktop приложение |
|---------|---------------------------|
| **Причина** | NW.js — desktop технология; фокус на глубокую интеграцию с браузером |
| **Альтернатива** | Companion mobile app в будущем (см. COMPETITOR_ANALYSIS.md) |
| **Пересмотреть когда** | Валидация PMF на desktop |

### Облачная синхронизация

| Решение | Локальное хранение данных |
|---------|---------------------------|
| **Причина** | Приватность пользователей, отсутствие backend инфраструктуры, сложность sync |
| **Альтернатива** | Export/Import данных вручную |
| **Пересмотреть когда** | Появится потребность в team features |

### Multi-profile поддержка

| Решение | Один профиль на установку |
|---------|---------------------------|
| **Причина** | Усложнение UX и архитектуры, низкий приоритет для MVP |
| **Альтернатива** | Несколько установок приложения |
| **Пересмотреть когда** | Запрос от значительной части пользователей |

### Поддержка Chrome Extensions

| Решение | Расширения не поддерживаются |
|---------|------------------------------|
| **Причина** | NW.js не поддерживает Chrome Extensions API; потребуется значительная работа |
| **Альтернатива** | Встроенные функции заменяют популярные расширения |
| **Пересмотреть когда** | Появится техническое решение от NW.js |

## 🐛 Известные проблемы и костыли

### Критические

| ID | Проблема | Описание | Workaround |
|----|----------|----------|------------|
| KB-001 | **electron-store в NW.js** | Используется `electron-store` для хранения данных, хотя это Electron-библиотека | Работает через shim, но может сломаться при обновлении |

### Высокий приоритет

| ID | Проблема | Описание | Workaround |
|----|----------|----------|------------|
| KB-002 | **Webview nodeintegration** | `<webview>` использует `nodeintegration`, что создаёт риски безопасности | Не загружать недоверенный контент |
| KB-003 | **Focus блокировка неполная** | Блокировка через `dom-ready` не предотвращает навигацию, только показывает overlay | Пользователь может закрыть overlay |

### Средний приоритет

| ID | Проблема | Описание | Workaround |
|----|----------|----------|------------|
| KB-004 | **Монолитный index.html** | Весь код (HTML, CSS, JS) в одном файле | Работает, но затрудняет поддержку |
| KB-005 | **Hardcoded blocked domains** | Список блокируемых сайтов захардкожен в коде | Требует пересборки для изменения |
| KB-006 | **Prompt для инсайтов** | Используется `prompt()` для ввода инсайтов — плохой UX | Заменить на inline UI |

### Низкий приоритет

| ID | Проблема | Описание | Workaround |
|----|----------|----------|------------|
| KB-007 | **Нет обработки ошибок загрузки** | Webview не показывает ошибки сети | Пользователь видит пустую страницу |
| KB-008 | **Нет истории навигации** | Кнопки назад/вперёд без визуальной индикации состояния | — |
| KB-009 | **Таймер сбрасывается при закрытии** | Focus таймер не сохраняется между сессиями | — |

## 🔧 Планы по рефакторингу

### Краткосрочные (следующий спринт)

- [ ] **Разделить index.html на модули** — отдельные файлы для CSS, JS
- [ ] **Заменить prompt() на inline UI** — модальное окно для ввода инсайтов
- [ ] **Вынести конфигурацию** — blocked domains, таймеры в отдельный config файл

### Среднесрочные (1-2 месяца)

- [ ] **Переход на безопасный webview** — отключить nodeintegration, использовать preload scripts
- [ ] **Заменить electron-store** — использовать localStorage или leveldb напрямую
- [ ] **Добавить error boundaries** — обработка ошибок загрузки страниц

### Долгосрочные (квартал)

- [ ] **Модульная архитектура** — разделение на компоненты (navigation, focus, insights)
- [ ] **TypeScript миграция** — типизация для большей надёжности
- [ ] **Unit тесты** — покрытие критической бизнес-логики
