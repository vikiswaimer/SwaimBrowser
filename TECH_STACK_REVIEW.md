# Обзор технологий для Swaim Browser

## 📍 Текущее состояние

Проект использует:
- **NW.js** — десктопная оболочка
- **Vanilla JavaScript** — без фреймворков
- **Webview** — для отображения веб-контента

### ⚠️ Проблемы текущего кода

```javascript
const nw = require('nw.gui');
const Store = require('electron-store'); // НЕСОВМЕСТИМО с NW.js!
```

`electron-store` — библиотека для Electron, она не работает в NW.js. Это нужно исправить.

## 🔧 Обзор фреймворков для десктопных приложений

### 1. Electron

| Параметр | Значение |
|----------|----------|
| **Размер приложения** | ~150-200 MB |
| **Использование RAM** | 100-300+ MB |
| **Язык** | JavaScript/TypeScript |
| **Зрелость** | Очень высокая (2013+) |
| **Кто использует** | VS Code, Slack, Discord, Figma, Notion, Arc Browser |

**Плюсы:**
- ✅ Огромная экосистема и сообщество
- ✅ Лучшая документация
- ✅ Множество готовых решений (auto-update, crash reporting)
- ✅ Отличная поддержка webview (`<webview>` или BrowserView)
- ✅ Нативные меню, трей, notifications
- ✅ Активная разработка (Chromium обновляется регулярно)
- ✅ AI-агенты знают Electron очень хорошо

**Минусы:**
- ❌ Большой размер приложения
- ❌ Высокое потребление памяти
- ❌ Каждое приложение тащит свой Chromium

**Вердикт для Swaim:** ⭐⭐⭐⭐⭐ **Рекомендуется**

### 2. NW.js (Node-Webkit)

| Параметр | Значение |
|----------|----------|
| **Размер приложения** | ~150-200 MB |
| **Использование RAM** | 100-300+ MB |
| **Язык** | JavaScript/TypeScript |
| **Зрелость** | Высокая (2011+) |
| **Кто использует** | Редко используется в новых проектах |

**Плюсы:**
- ✅ Node.js доступен прямо в DOM (без IPC)
- ✅ Проще архитектура (нет main/renderer процессов)
- ✅ Можно запускать Chrome Apps
- ✅ Поддержка Source Code Protection

**Минусы:**
- ❌ Меньше сообщество и экосистема
- ❌ Хуже документация
- ❌ Меньше готовых библиотек
- ❌ AI-агенты хуже знают NW.js
- ❌ Webview API менее развит, чем в Electron
- ❌ Меньше обновлений безопасности

**Вердикт для Swaim:** ⭐⭐⭐ **Не рекомендуется**

---

### 3. Tauri

| Параметр | Значение |
|----------|----------|
| **Размер приложения** | ~3-10 MB (!!) |
| **Использование RAM** | 30-80 MB |
| **Язык** | Rust (бэкенд) + JS/TS (фронтенд) |
| **Зрелость** | Средняя (v1.0 в 2022, v2.0 в 2024) |
| **Кто использует** | Новые проекты, Codeium, некоторые инструменты |

**Плюсы:**
- ✅ **Минимальный размер** — в 20-30 раз меньше Electron
- ✅ **Низкое потребление RAM** — в 3-5 раз меньше Electron
- ✅ Использует системный WebView (WebView2/WebKit)
- ✅ Отличная безопасность (Rust)
- ✅ Быстрый старт приложения
- ✅ Активная разработка, растущее сообщество

**Минусы:**
- ❌ **Webview ограничен** — нет полноценного контроля как в Electron
- ❌ Зависимость от системного WebView (разное поведение на разных ОС)
- ❌ Нужно знать Rust для нативных функций
- ❌ Меньше готовых библиотек
- ❌ AI-агенты знают Tauri хуже, чем Electron
- ❌ **Критично для браузера:** сложнее управлять webview, history, extensions

**Вердикт для Swaim:** ⭐⭐ **Не подходит для браузера**

> ⚠️ Tauri отлично подходит для обычных приложений, но **для браузера критична работа с webview**, которая в Tauri ограничена.

## 🎯 Рекомендации для Swaim Browser

### Рекомендуемый стек

```
┌─────────────────────────────────────────────┐
│                 Swaim Browser                │
├─────────────────────────────────────────────┤
│  UI Layer:         React + TypeScript        │
│  UI Components:    shadcn/ui (Radix-based)   │
│  State:            Zustand                   │
│  Desktop Runtime:  Electron                  │
│  Storage:          electron-store            │
│  Build:            Vite + electron-builder   │
└─────────────────────────────────────────────┘
```

### Почему Electron + React?

1. **Electron:**
   - Лучшая поддержка webview для браузера
   - BrowserView API позволяет полный контроль над навигацией
   - Огромная экосистема (auto-update, crash reporting)
   - AI-агенты знают Electron отлично
   - Используется в Arc Browser (наш конкурент)

2. **React:**
   - AI-агенты генерируют качественный React-код
   - shadcn/ui даёт красивые компоненты из коробки
   - TypeScript обеспечивает type safety
   - Легко масштабировать

3. **Zustand:**
   - Простой state management
   - Минимальный boilerplate
   - Хорошо работает с React

### Альтернативный стек (если важна производительность)

```
┌─────────────────────────────────────────────┐
│                 Swaim Browser                │
├─────────────────────────────────────────────┤
│  UI Layer:         Svelte + TypeScript       │
│  UI Components:    Skeleton UI               │
│  State:            Svelte stores             │
│  Desktop Runtime:  Electron                  │
│  Storage:          electron-store            │
│  Build:            Vite + electron-builder   │
└─────────────────────────────────────────────┘
```

**Плюсы:** Меньше размер bundle, быстрее рендеринг
**Минусы:** Меньше готовых компонентов, AI хуже помогает

---

## 🚫 Почему НЕ оставаться на NW.js + Vanilla JS?

1. **NW.js:**
   - Меньше документации и примеров
   - AI-агенты хуже генерируют код
   - `electron-store` не работает в NW.js (нужен `nw-store` или своё решение)
   - Меньше обновлений безопасности

2. **Vanilla JS:**
   - Сложно поддерживать при росте приложения
   - Нет компонентного подхода
   - Много repetitive кода
   - Сложнее для AI генерировать консистентный код

---

## 🚫 Почему НЕ Tauri?

Tauri отличен для обычных приложений, но **не для браузера**:

1. **Ограниченный webview:**
   - Используется системный WebView (Safari на macOS, WebView2 на Windows)
   - Разное поведение на разных ОС
   - Нет полного контроля над навигацией, history, cookies

2. **Нет BrowserView:**
   - Electron BrowserView позволяет создавать настоящий браузер
   - В Tauri это невозможно или очень сложно

3. **Extensions:**
   - Если захотим поддержку расширений — только Chromium (Electron)

## ✅ Итоговое решение

### Для Swaim Browser рекомендуется:

| Компонент | Технология | Причина |
|-----------|------------|---------|
| Runtime | **Electron** | Лучший webview, экосистема, AI-friendly |
| UI | **React + TypeScript** | Масштабируемость, AI-friendly |
| Components | **shadcn/ui** | Красивые, кастомизируемые |
| State | **Zustand** | Простой, эффективный |
| Storage | **electron-store** | Простой key-value storage |
| Build | **Vite** | Быстрая сборка |
| Package | **electron-builder** | Кросс-платформенность |
